# feat: Stop committing generated `index.json` (Pages-only)

Issue: #9

## Background

`index.json` is a generated artifact:

- `build-index.yml` regenerates it on **every push to `main`** and publishes it (plus the JSON
  Schemas) to **GitHub Pages**. The MulmoClaude backend reads it from that stable Pages URL.
- By design it is **never committed back to `main`** (R2 — no bot-commits to main).
- Nothing in the repo consumes the committed copy: `grep -rn "index.json" scripts test` returns
  only the builder that *writes* it.

Despite that, `pr-validate` runs `build-index.mjs --check`, which enforces that the **committed**
`index.json` matches the source content. Consequences:

1. **Redundant friction** — contributors must regenerate and commit a file no one reads. PR #8
   failed CI for exactly this (jma-weather content changed in the final commit without rerunning
   `build-index`).
2. **Cross-PR contamination** — PR #8 merged with a stale committed `index.json`, and
   `build-index.yml` never commits back, so `main`'s committed `index.json` is **stale now**
   (`build-index.mjs --check` fails on `main`). Because `--check` validates the *whole* index, the
   **next unrelated PR fails validate through no fault of its own**.

## Decision

Stop tracking `index.json`. Keep it generated-and-published only. Keep guarding each collection's
`manifest.json` — those are committed and fetched from git by the host, so they must stay correct.

## Changes

- **`.gitignore`** — ignore root `/index.json` (anchored; never matches collection files).
- **`git rm --cached index.json`** — stop tracking; the file stays on disk (gitignored) and is
  regenerated by `build-index.yml` for Pages.
- **`scripts/build-index.mjs`**
  - `--check` (`staleArtifacts`) checks **manifests only**; drop the `index.json` branch and the
    now-unused `sansTimestamp` helper / `readFileSync` import.
  - Write mode still writes `index.json` (for Pages) **and** manifests — unchanged.
  - Update header/usage/success messages to say manifests are guarded; `index.json` is generated
    for publishing, not committed.
- **`.github/workflows/pr-validate.yml`** — keep the `build-index.mjs --check` step (now a manifest
  guard); rename it and fix its comment.
- **`README.md` / `CONTRIBUTING.md`** — clarify `index.json` is generated-and-published (not
  committed) while manifests are committed; update the commands list.
- **`package.json`** — rename `check-index` → `check-manifests` (only referenced by README).

## Out of scope (kept committed + CI-guarded)

`manifest.json` per collection — part of the importable bundle the host fetches by raw URL.

## Verification

- `node scripts/build-index.mjs --check` passes on this branch (manifests fresh).
- `node --test` passes.
- `node scripts/validate.mjs` passes.
- `git ls-files index.json` returns nothing (untracked).
- `build-index.yml` still produces `index.json` + schemas into `_site` (logic unchanged).
